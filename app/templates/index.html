FAST FOOD FAST API

Welcome dear developer

Welcome to the fast-food-fast-api, below are some few notes you need to take to use the api at the current stage of development

Getting started

To get started you need to have one of both of the following

  1. Atleast one user to test how the orders work
    To create a user use the following endpoints
    1. POST to the endpoint /api/v1/users

    Remember just a single user is enough for testing purposes, you can create as many users as you want but only the first user is used for testing purposes

  2. Atleast one product for the user to order
    To create a product use the following endpoints
    1. POST to the endpoint /api/v1/products

  3. To order you must also have atleast one destination address in the database
    To create an address use the endpoint below
    1. POST to endpoint /api/v1/addresses

    Remember since this is just for testing, so the address will be assigned to the test user who is the only user we've used for testing

After the above are set up

To make an order the following fields are required fields

  • order_id
  • address
  • items

    Note that the items should be comma separated list of item ids from the previously added products if you're not sure of the id please double check through the route /api/v1/products to get the ids that exist

Thank you so much and nice time using the API
